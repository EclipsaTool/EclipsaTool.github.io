Git基础命令速查手册

  2020-08-15   笔记   计算机 git  评论 字数统计: 2019字 阅读时长: 5分

Git基础命令速查手册

作者：eclipsa

摘要：git常用指令分类汇总。文中使用的系统为ubuntu。

[toc]

常用命令与文件位置

安装Git

ubuntu：sudo apt-get install git

获取帮助

git help
git help <指令>

分类

基本配置

邮箱信息

配置全局邮箱信息，针对每个仓库。

git config --global user.name "John Doe"
git config --global user.email johndoe@example.com

配置本地邮箱信息，针对个别仓库。

git config --local user.name "John Doe"
git config --local user.email johndoe@example.com

检查配置

git config --list

.gitignore

以#开头的git会认为是注释。

默认匹配模式会自动递归整个工作目录。使用标准glob模式1。

匹配模式以/开头可以忽略指定目录。

匹配模式以/结尾可以忽略指定的文件夹名。(会递归，所有同名文件夹都忽略)

以!开头可以指定跟踪某些匹配模式匹配的内容。(相当于补丁)

# 忽略所有的 .a 文件
*.a

# 但跟踪所有的 lib.a，即便你在前面忽略了 .a 文件
!lib.a

# 只忽略当前目录下的 TODO 文件，而不忽略 subdir/TODO
/TODO

# 忽略任何目录下名为 build 的文件夹
build/

# 忽略 doc/notes.txt，但不忽略 doc/server/arch.txt
doc/*.txt

# 忽略 doc/ 目录及其所有子目录下的 .pdf 文件
doc/**/*.pdf

获取内置帮助。

man gitignore

工具配置

git config --global core.editor可以指定git默认用的文本编辑器。

本地仓库

建立仓库

建立本地空白仓库

cd /home/eclipsa/example_git_workspace
git init

文件操作

查看文件状态

git status
git stauts -s

准确添加文件

git add <文件名>

撤销添加文件

git reset HEAD <文件名>

查看文件异同

git diff
git diff --cached

提交文件

git commit
git commit -m "修改注释”
git commit <文件名>

git commit -a <文件名>

重新提交

git commit --amend

查看缓存区文件列表

git ls-files

移除文件

git rm <文件名>
git rm --cached <文件名>

移动/重命名文件

git mv <文件名> <新文件名>

工作区单个文件还原上次提交状态

git checkout -- <文件名>

清理文件

用于清理git中未跟踪的软件(.gitignore也算跟踪)。

git clean
git clean -x

日志操作

查看日志

git log

git log -p -2
git log -patch -2

git log --stat

git log有用的参数

参数 作用
–graph 在日志旁以 ASCII 图形显示分支与合并历史。
-p 按文件补丁形式显示。
–name-only 仅在提交信息后显示已修改的文件清单。
–name-status 显示新增、修改、删除的文件清单。
–oneline --pretty=oneline --abbrev-commit 合用的简写。一行显示。
–author=”” 限制作者。
–since, –after 指定时间。
–grep 仅显示提交说明中包含指定字符串的提交。

远程仓库

建立仓库

克隆远程仓库

(git clone支持多种通讯协议。)

git clone <url>

git clone ssh://[user@]host.xz[:port]/path/to/repo.git/
git clone git://host.xz[:port]/path/to/repo.git/
git clone http[s]://host.xz[:port]/path/to/repo.git/
git clone ftp[s]://host.xz[:port]/path/to/repo.git/

为本地仓库添加远程仓库

git remote add <仓库服务器别名> <url>

仓库状态

git remote
git remote -v

git remote show <远程仓库服务器名>

仓库操作

获取远程仓库

git fetch <远程仓库服务器名>

推送本地仓库

git push <远程仓库服务器名> <推送本地版本库分支名>
git push <远程仓库服务器名> <推送本地版本库分支名>:<远程仓库分支名>

同步远程仓库

git pull

重命名远程仓库

git remote rename <远程仓库服务器名> <远程仓库服务器新名>

删除远程仓库

git remote remove <远程仓库服务器名>
git remote rm <远程仓库服务器名>

分支

分支状态

查看分支

git branch
git branch -v
git branch --no--merged
git branch --merged
git branch --merged <分支名>
git branch -vv

分支操作

创建分支

git branch <分支名>

创建并切换分支

git checkout -b <分支名>

切换分支

git checkout <分支名>

合并分支

git merge <分支名>

删除分支

git branch -d <分支名>

创建跟踪分支

git checkout -b <本地分支名> <远程服务器名>/<远程分支名>
git checkout --track <远程服务器名>/<远程分支名>

设置/修改上游分支

git branch -u <远程服务器名>/<远程分支名>

下载分支版本库

git fetch --all

删除远程分支

git pull <远程服务器名> --delete <分支名>

贮藏

贮藏状态

列出贮藏

git stash list

贮藏操作

创建贮藏

git stash
git stash push

应用贮藏

--index可以让git在应用贮藏时，同时应用到工作目录和缓存区，而非只是工作目录。

git stash apply
git stash apply stash@{贮藏号}
git stash apply --index

移除贮藏

git stash drop
git stash drop stash@{贮藏号}

应用贮藏同时移除贮藏

git stash pop

从贮藏创建分支

git stash branch <分支名>
git stash branch <分支名> stash@{贮藏号}

版本库标签

标签状态

列出全部标签

git tag

显示标签信息

git tag show

查看标签快照

git checkout <标签名>

标签操作

给当前版本打标签

git tag -a <标签名>

补打标签

git tag -a <标签名> <部分校验和/完整校验和>

推送标签

git pull <分支名> <标签名>
git pull <分支名> --tags

删除标签

git tag -d <标签名>

删除服务器标签

git pull <远程仓库服务器名> :refs/tags/<标签名>
git pull <远程仓库服务器名> --delete <标签名>

  1. shell所使用的，简化来的正则表达式。星号*匹配零个或多个任意字符。问号?匹配一个任意字符。[abc]匹配任意一个方括号中的字符。[0-9]匹配0到9的任意一个数字。**匹配任意位于中间的目录 ↩︎
